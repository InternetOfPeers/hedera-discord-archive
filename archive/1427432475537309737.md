## Message 1434227941285564647

By @Adrian | Kabila (adriankbl#0 291665472988905472)
at *2025-11-01 17:09:47.869 UTC*

```txt
I’ve been discussing this issue with **LimeChain** for quite some time. They recently released `Client.forMainnetAsync()`, which syncs available nodes via the REST API. However, in production environments, we obviously can’t rely on the public mirror node. We use the official Hedera endpoint, but here’s the problem:

If a DApp is configured to use node 0.0.3—either because it’s using an older SDK version or its transaction list includes nodes 0.0.3 through 0.0.9—and Reown selects 0.0.3, which happens to be down, while on the wallet side you’re checking Hedera’s API for healthy nodes, the transaction will fail with an `INVALID_NODE_ID` error , since the Wallet side does not have the node `0.0.3` setted up in his `Client`.

The best solution is to update the spec and fix this issue so that DApps can optionally delegate node selection to the wallet side, ensuring more reliable execution.
```

```txt
What I’m referring to is that if the DApp sets a specific node in the transaction and the wallet’s signer doesn’t have that node in its list, an error occurs — if I’m not mistaken, that’s what triggers the `INVALID_NODE_ACCOUNT` issue.

In our case, this happened occasionally, for example, with the following flow:
    1.    Kabila updated to the latest SDK version, which included a new node in the client.
    2.    We used that new node to submit a transaction.
    3.    The HashPack client, during initialization, didn’t have that node in its configuration when signing.
    4.    The result was an `INVALID_NODE_ACCOUNT` error.

I’m explaining this from memory, after running many tests and having several discussions with both Hashgraph and LimeChain about it. In our case, for example, we never use `Client.forMainnet()`. Instead, we use `Client.forNetwork()` to ensure that the nodes set in the transaction — and, in the case of HashPack (which is required to receive only one node due to Reown’s implementation) — are always healthy.
```
